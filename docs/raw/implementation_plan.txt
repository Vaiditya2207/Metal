# Metal  Strategic Phase Plan for a macOS-Native Browser Engine

A phased roadmap to build a high-performance, macOS-exclusive browser engine from scratch using **Zig**, **Metal**, and **Apple Silicon** optimizations. Each phase is designed to produce a working, testable artifact before the next begins. Dependencies flow strictly downward.

---

## Guiding Principles

| Principle | Rationale |
|---|---|
| **Vertical slices** | Every phase ends with something you can see, run, or measure. No "invisible infrastructure" phases. |
| **Hardest-risk-first** | The rendering pipeline and JS binding are the two riskiest unknowns  they come early. |
| **Incremental spec coverage** | Start with a trivial HTML/CSS subset and widen the spec surface over time. |
| **Test-as-you-go** | Each phase includes its own verification criteria. |

## Phase 0  Toolchain Bootstrap & Project Skeleton
**Goal:** A Zig project that compiles, links against macOS system frameworks, and runs an empty executable.

### Deliverables
- `build.zig` configured with:
  - Zig `0.14+` targeting `aarch64-macos`
  - C/ObjC interop flags (`@cImport` of `<Cocoa/Cocoa.h>`, `<Metal/Metal.h>`)
  - Linking against `AppKit`, `Metal`, `MetalKit`, `QuartzCore`
- Project directory structure:

```
metal/
 build.zig
 build.zig.zon          # package manifest
 src/
    main.zig           # entry point
    platform/           # macOS / ObjC bridge
    dom/                # HTML tokenizer, DOM tree
    css/                # CSS parser, style resolver
    layout/             # box model, layout tree
    render/             # Metal renderer, MSDF text
    js/                 # JS engine bindings
    net/                # networking, resource loader
    ipc/                # process isolation, shared memory
    ui/                 # tab bar, URL bar, chrome
    devtools/           # native developer tools
    ml/                 # Core ML / MLX integration
 resources/
     shaders/            # Metal shader sources (.metal)
     fonts/              # default system font atlas
```
