## 1. Introduction

### 1.1 Purpose
This document specifies the functional, non-functional, and system requirements for **Metal**, a high-performance web browser engine built exclusively for macOS on Apple Silicon. It serves as the single source of truth for what the system must do.

### 1.2 Scope
Metal is a complete browser application comprising:
- An HTML/CSS parsing and layout engine
- A GPU-accelerated rendering pipeline (Apple Metal API)
- A JavaScript execution environment (JavaScriptCore + QuickJS)
- A networking stack (via NSURLSession)
- A native macOS UI (AppKit + Metal)
- On-device ML for predictive caching and AI assistance
- Native developer tools

### 1.3 Definitions & Acronyms

| Term | Definition |
|---|---|
| RSS | Resident Set Size  physical RAM consumed |
| UMA | Unified Memory Architecture |
| MSDF | Multi-Channel Signed Distance Field |
| IPC | Inter-Process Communication |
| DOM | Document Object Model |
| JSC | JavaScriptCore |
| QoS | Quality of Service (macOS thread priority) |
| ANE | Apple Neural Engine |

## 4. Non-Functional Requirements

### NFR-1: Performance

| ID | Requirement | Target |
|---|---|---|
| NFR-1.1 | Frame render time | < 4 ms at 120 Hz (ProMotion) |
| NFR-1.2 | Initial page load (cached, simple page) | < 200 ms |
| NFR-1.3 | Tab switch latency | < 50 ms |
| NFR-1.4 | JS startup (JSC context creation) | < 5 ms |
| NFR-1.5 | JS startup (QuickJS context creation) | < 300 s |
| NFR-1.6 | Cold launch to window visible | < 500 ms |

### NFR-2: Memory

| ID | Requirement | Target |
|---|---|---|
| NFR-2.1 | Browser idle (1 blank tab) | < 30 MB RSS |
| NFR-2.2 | Per-tab overhead (simple page) | < 15 MB RSS |
| NFR-2.3 | 50 tabs open (mixed complexity) | < 1 GB RSS |
| NFR-2.4 | Background tab (compressed) | < 2 MB RSS |
| NFR-2.5 | Zero memory leaks on tab close | Arena freed completely |

### NFR-3: Security

| ID | Requirement | Target |
|---|---|---|
| NFR-3.1 | Process isolation between tabs | Separate OS processes |
| NFR-3.2 | App Sandbox for renderer processes | macOS entitlements enforced |
| NFR-3.3 | No cross-origin data leaks | Same-origin policy enforced |
| NFR-3.4 | All ML inference on-device | Zero data exfiltration |

### NFR-4: Compatibility

| ID | Requirement | Target |
|---|---|---|
| NFR-4.1 | Minimum macOS version | macOS 14 Sonoma |
| NFR-4.2 | Minimum hardware | Apple M1 |
| NFR-4.3 | Display support | Standard + Retina + ProMotion |

### NFR-5: Accessibility

| ID | Requirement | Target |
|---|---|---|
| NFR-5.1 | VoiceOver support | Full navigation |
| NFR-5.2 | Keyboard-only navigation | All features accessible |

## 6. Constraints

| Constraint | Description |
|---|---|
| **macOS-only** | No cross-platform support. Deep Apple API integration is a feature, not a limitation. |
| **Apple Silicon only** | No Intel Mac support. Enables UMA, ANE, and Metal 3 assumptions. |
| **No WebKit/Blink reuse** | Entire rendering engine is custom-built. WebKit is used only indirectly (JSC framework). |
| **Zig maturity** | Zig is pre-1.0. API breakage between releases is expected. Pin to a specific version. |

## 7. Acceptance Criteria (MVP  Phase 08)

The browser is considered **MVP-complete** when:

- [ ] Can navigate to a URL and render HTML + CSS with text and images
- [ ] JavaScript executes and can manipulate the DOM
- [ ] Scrolling is smooth at 120 Hz
- [ ] Links are clickable and navigate correctly
- [ ] Text input works in `<input>` fields
- [ ] RSS stays under 50 MB for a single simple page
- [ ] Tab close fully frees all associated memory
- [ ] No crashes on the top 10 test pages
