Metal Coding Style Guide

1. Memory Management
- Every allocator MUST be explicit.
- Use std.heap.ArenaAllocator for objects with a shared lifecycle.
- Avoid page_allocator directly; use it only at the high level.

2. Error Handling
- Use Zig's error union types (!T) for all fallible operations.
- Avoid catch unreachable.

3. Naming Conventions
- Types/Structs: PascalCase.
- Functions: camelCase.
- Variables: snake_case.
- Files: snake_case.

4. Concurrency & QoS
- Use QOS_CLASS_USER_INTERACTIVE for the main loop.
- Use QOS_CLASS_BACKGROUND for non-critical tasks.
- Leverage Mach ports for IPC.

5. Configurability
- No hardcoded UI constants.
- Use JSON or TOML for all application and engine settings.
- Ensure all parameters are adjustable without recompilation.

6. Security
- All external input (HTML, CSS, JS, network data) is UNTRUSTED.
- Enforce configurable bounds on all dynamic structures (tree depth, node count, attribute sizes).
- Use arena allocators per document for O(1) cleanup.
- Never use catch unreachable on untrusted data paths.
- Graceful degradation: malformed input must never panic or leak memory.
- All parsing limits must be loaded from the configuration system at startup.
